Welcome to Mission Planner

Overview

This Angular 16 application provides a simplified mission planning capability, empowering operators to create and visualize robot movement. It leverages Angular Material for UI components and follows best practices for modular and maintainable code.

Application Structure

  • src/app: Contains the main application code.
    • app.component.ts: The root component of the application.
    • app.module.ts: The main module that bootstraps the application.
    • app-routing.module.ts: Defines the routes for the application.
  • src/app/features: Contains feature components.
    • home: Home feature standalone component.
    • planner: Planner feature standalone component.
    • player: Player feature standalone component.
  • src/app/shared: Contains shared components, such as the navigation bar, page-not-found component.
    • components: Reusable components shared across the application.
  • src/assets: Contains static assets such as images.
  • src/environments: Configuration for different environments (development, production).

Getting Started

Prerequisites

  • Node.js (v14 or later)
  • Angular CLI (v16 or later)

Installation

  1. Clone the repository: 
    git clone https://github.com/teodora2004/mission-planner
cd mission-planner
  2. Install dependencies: 
    npm install

Running the Application

  1. Serve the application: 
    ng serve

    The application will be served at http://localhost:4200/. Open this URL in your web browser to view the application.

Building the Application

  1. Build for production: 
    ng build --prod

    This command will create a dist/ directory containing the compiled application.

Additional Commands

  • Lint the code: 
    ng lint
  • Run unit tests: 
    ng test

JSON Server Integration

Overview

To store the points, a JSON server is used as a mock backend. This allows for easy testing and development of the application without the need for a real backend.

Setup

  1. Install JSON Server: 
    npm install json-server --save-dev
  2. Start JSON Server: 
    npm run start:json-server

    The JSON server will run at http://localhost:3000/.

Configuration

The JSON server URL can be configured via the environment file. Update the environment files as follows:

// src/environments/environment.ts
export const environment = {
  production: false,
  apiUrl: 'http://localhost:3000'
};

// src/environments/environment.prod.ts
export const environment = {
  production: true,
  apiUrl: 'http://localhost:3000'
};

In your services, you can use the apiUrl from the environment file to make HTTP requests to the JSON server.
